# Выполнение хакатона «Автономные дроны и бортовой ИИ» | MISIS Robotics Week
В данном репозитории представлены все файлы решения команды misis_team для хакатон-интенсива «Автономные дроны и бортовой ИИ» от компании «Сверх» в рамках мероприятия MISIS Robotics Week!

# Описание задания хакатона:
Хакатон-интенсив представлял собой соревнование, целью которого была разработка программного модуля автономной навигации для образовательного «Обрик» в условиях, приближенных к реальной задаче автоматизации складских помещений.
Основная миссия для участников была разбита на две ключевые задачи, которые необходимо было выполнить в едином полетном задании:
1. Автономная навигация и картирование (SLAM): В полетной зоне размером 3х3х3 метра организаторами динамически выстраивался лабиринт из вертикальных перегородок высотой 1,2 метра. Конфигурация лабиринта (топология коридоров и тупиков) заранее была неизвестна участникам. Задача БВС — используя только бортовые сенсоры (лазерный дальномер, камеру, датчики оптического потока), построить 2D-карту этого неизвестного лабиринта в реальном времени и автономно перемещаться внутри него, стартуя из зоны старта и достигая зоны финиша.
2. Бортовой ИИ и поиск объектов: Во время движения по лабиринту БВС должен был в режиме реального времени, с помощью нейросетей семейства YOLO (v5/v8/v11), обнаруживать и классифицировать физические объекты, имитирующие цели инвентаризации на складе. В лабиринте в случайных местах были размещены два типа объектов: «Апельсин» и «Мишка». Задача дрона — не только найти их, но и идентифицировать, выведя в логи название обнаруженного объекта.

> [!IMPORTANT]
> [«Обрик»](https://docs.sverh.tech/ru/) — это образовательная платформа программируемого квадрокоптера для обучения роевому взаимодействию, компьютерному зрению и автономной навигации, на базе популярных открытых компонентов с документацией и библиотеками. Набор включает в себя полётный контроллер с прошивкой PX4, Raspberry Pi 4 Compute Module (CM4) с платой ввода-вывода в качестве управляющего бортового компьютера, модуль камеры для реализации полетов с использованием компьютерного зрения, а также набор датчиков. Вес платформы до 300 грамм, что делает дрон безопасным для использования в классах и лабораториях школ, колледжей и вузов.

# Описание работы кода для камеры:
Все файлы для работы камеры лежат в папке _ros2_camera_code_.

1. _camera_node.py_ — Узел захвата видеопотока. Этот модуль транслирует видео с камеры в среду ROS 2, используя GStreamer или V4L2 для работы на Raspberry Pi 5. Кадры в разрешении 640x480 преобразуются через CvBridge в сообщения _sensor_msgs/Image_ и публикуются в топик _/camera/image_raw_.
2. _ai_node.py_ — Узел нейросетевой детекции объектов. Узел использует модель YOLOv8 для поиска объектов, анализируя кадр в четырех ориентациях (0°–270°) для компенсации крена дрона. После фильтрации дубликатов алгоритмом NMS данные о целях передаются в JSON-формате через топик _/drone/detection_data_, а снимки сохраняются в папку _detections_.
3. _aruco_ai_code.py_ — Узел навигации по маркерам. Компонент вычисляет 3D-дистанцию и вектор направления до ArUco-маркеров размером 5 см методом SolvePnP. Координаты смещения и расстояние публикуются в топик _aruco/vector_data_, а для отладки формируется изображение с осями координат в топике _aruco/debug_image_.

# Команда проекта:
[Голышев Юрий Александрович ](https://github.com/wodocanal)— отвечал за интеграцию ROS 2 с полетным контроллером, адаптацию алгоритма SLAM для построения карты лабиринта и реализацию логики автономного полета с аварийной защитой.

[Симакова Анастасия Александровна ](https://github.com/fismay)— руководила разработкой, отвечала за интеграцию YOLO для распознавания объектов, настройку пайплайна компьютерного зрения и калибровку камеры, координировала работу команды.

[Ромазанов Артём Валентинович](https://github.com/Artem-Romazanov)— обеспечивал техническую подготовку: настройку Ubuntu и ROS 2, тестирование сенсоров, помогал с настройкой окружения для ML и готовил демонстрационные материалы.

[Тамбовцев Савва Викторович](https://github.com/CardlnaI)— занимался дообучением YOLO, тестировал качество распознавания,настройку пульта с Kill-switch, ассистировал при полетных тестах и анализировал логи.
