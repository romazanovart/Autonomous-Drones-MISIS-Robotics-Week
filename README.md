# Описание задания хакатона:
Хакатон-интенсив представлял собой соревнование, целью которого была разработка программного модуля автономной навигации для образовательного «Обрик» в условиях, приближенных к реальной задаче автоматизации складских помещений.
Основная миссия для участников была разбита на две ключевые задачи, которые необходимо было выполнить в едином полетном задании:
1. Автономная навигация и картирование (SLAM): В полетной зоне размером 3х3х3 метра организаторами динамически выстраивался лабиринт из вертикальных перегородок высотой 1,2 метра. Конфигурация лабиринта (топология коридоров и тупиков) заранее была неизвестна участникам. Задача БВС — используя только бортовые сенсоры (лазерный дальномер, камеру, датчики оптического потока), построить 2D-карту этого неизвестного лабиринта в реальном времени и автономно перемещаться внутри него, стартуя из зоны старта и достигая зоны финиша.
2. Бортовой ИИ и поиск объектов: Во время движения по лабиринту БВС должен был в режиме реального времени, с помощью нейросетей семейства YOLO (v5/v8/v11), обнаруживать и классифицировать физические объекты, имитирующие цели инвентаризации на складе. В лабиринте в случайных местах были размещены два типа объектов: «Апельсин» и «Мишка». Задача дрона — не только найти их, но и идентифицировать, выведя в логи название обнаруженного объекта.

> [!IMPORTANT]
> [«Обрик»](https://docs.sverh.tech/ru/) — это образовательная платформа программируемого квадрокоптера для обучения роевому взаимодействию, компьютерному зрению и автономной навигации, на базе популярных открытых компонентов с документацией и библиотеками. Набор включает в себя полётный контроллер с прошивкой PX4, Raspberry Pi 4 Compute Module (CM4) с платой ввода-вывода в качестве управляющего бортового компьютера, модуль камеры для реализации полетов с использованием компьютерного зрения, а также набор датчиков. Вес платформы до 300 грамм, что делает дрон безопасным для использования в классах и лабораториях школ, колледжей и вузов.

# Описание работы кода для камеры:
Все файлы для работы камеры лежат в папке [_ros2_camera_code_](https://github.com/Artem-Romazanov/Autonomous-Drones-MISIS-Robotics-Week/tree/main/ros2_camera_code).

1. [_camera_node.py_](https://github.com/Artem-Romazanov/Autonomous-Drones-MISIS-Robotics-Week/blob/main/ros2_camera_code/camera_node.py) — Узел захвата видеопотока. Этот модуль транслирует видео с камеры в среду ROS 2, используя GStreamer или V4L2 для работы на Raspberry Pi 4. Кадры в разрешении 640x480 преобразуются через CvBridge в сообщения _sensor_msgs/Image_ и публикуются в топик _/camera/image_raw_.
2. [_ai_node.py_](https://github.com/Artem-Romazanov/Autonomous-Drones-MISIS-Robotics-Week/blob/main/ros2_camera_code/ai_node.py) — Узел нейросетевой детекции объектов. Узел использует модель YOLOv8 для поиска объектов, анализируя кадр в четырех ориентациях (0°–270°) для компенсации крена дрона. После фильтрации дубликатов алгоритмом NMS данные о целях передаются в JSON-формате через топик _/drone/detection_data_, а снимки сохраняются в папку _detections_.
3. [_aruco_ai_code.py_](https://github.com/Artem-Romazanov/Autonomous-Drones-MISIS-Robotics-Week/blob/main/ros2_camera_code/aruco_ai_code.py) — Узел навигации по маркерам. Компонент вычисляет 3D-дистанцию и вектор направления до ArUco-маркеров размером 5 см методом SolvePnP. Координаты смещения и расстояние публикуются в топик _aruco/vector_data_, а для отладки формируется изображение с осями координат в топике _aruco/debug_image_.

# Описание работы кода для камеры:
Все файлы для управления автономным полетом лежат в папке [_drone_navigation_](https://github.com/Artem-Romazanov/Autonomous-Drones-MISIS-Robotics-Week/tree/main/drone_navigation).

1. [_navigation_node.py_](https://github.com/Artem-Romazanov/Autonomous-Drones-MISIS-Robotics-Week/blob/main/drone_navigation/navigation_node.py) — Узел низкоуровневого управления позицией. Модуль реализует логику полета по жестко заданным путевым точкам (взлет, движение «змейкой», посадка), отслеживая текущие координаты через подписку на топик _/odom_. Узел самостоятельно вычисляет евклидово расстояние до цели и переключает этапы миссии при достижении заданного радиуса допуска (_tolerance_), непрерывно транслируя целевые координаты в топик _/setpoint_position/local_.
2. [_mission_script.py_](https://github.com/Artem-Romazanov/Autonomous-Drones-MISIS-Robotics-Week/blob/main/drone_navigation/mission_script.py) — Скрипт высокоуровневого выполнения миссий. В отличие от контроллера, данный модуль работает через сервисы (_/navigate_, _/land_), отправляя команды перемещения в системе координат _body_ (относительно текущего положения дрона). Модуль включает многопоточную обработку для фонового сбора телеметрии (заряд батареи, состояние моторов) и выполняет последовательный алгоритм полета по сложной траектории с программными задержками между точками.
